================================================================
第一部分 ★ 对外收单 API + 支付全链路（epay 核心价值，我们最缺）
================================================================
说明：这是易支付的心脏。epay 有 V1(旧,MD5签名) 和 V2(新,RSA签名) 两套对外协议，
      商户系统靠它们下单/查单/退款/代付。我们目前 /api/pay/* 整个不存在。

【下单 / 收银】
  ⬜/⬜  V1 页面下单  submit.php（GET 跳转到收银台/直接支付）
  ⬜/⬜  V1 API下单   mapi.php（返回 JSON，商户自渲染）
  ⬜/⬜  V2 下单      pay/submit、pay/create（RSA 签名，REST 风格）
  ⬜/⬜  收银台聚合页 cashier.php（无 type 时展示可选支付方式）
  ⬜/⬜  支付中间页   includes/pages/（扫码页/H5/JSAPI/公众号，按插件返回 type 渲染）
        —— 我们后端 internal/channel 有 interface 抽象，但零渲染、零下单入口。

【查单 / 退款】
  ⬜/⬜  V1 单笔查询  api.php?act=query
  ⬜/⬜  V1 批量查询  api.php?act=orders
  ⬜/⬜  V1 退款      api.php?act=refund
  ⬜/⬜  V2 查单/退款/退款查询  pay/query、pay/refund、pay/refundquery

【回调 / 通知（最关键的异步链路）】
  ⬜/⬜  第三方回调路由  /pay/notify/{trade_no}/ → 插件 notify() 验签
  ⬜/⬜  订单状态幂等更新 Payment::processOrder（防重复通知）
  ⬜/⬜  订单入账        functions.processOrder（按 tid 入账：普通/充值/买组/保证金）
  ⬜/⬜  商户异步通知     creat_callback + do_notify（回商户 notify_url，判 success）
  ⬜/⬜  通知失败重试     进 cron 队列(1/3/20分/1/2时 多档重试)
  ⬜/⬜  同步跳转 return  支付完成跳回商户 return_url

【商户信息 / 代付 API】
  ⬜/⬜  V2 商户信息     merchant/info
  ⬜/⬜  V2 商户订单     merchant/orders
  ⬜/⬜  V2 代付提交/查询/凭证/余额  transfer/submit|query|proof|balance

【签名 / 安全】
  ⬜  MD5 签名验签（V1：pid+key，ksort 去空去 sign）
  ⬜  RSA SHA256 签名验签（V2：平台私钥签、商户公钥验）
  ⬜  timestamp 防重放（V2，±300s）
  ⬜  下单幂等（订单号 + 渠道流水号 唯一键）
  ⬜  回调金额/状态二次校验防篡改
  —— 全部未实现。这是接后端阶段 A 的主战场。

   这条链一通，0538pay 才从"支付管理界面"变成"支付系统"。

   → D 商户中心接后端 → E 定时任务+收尾。
