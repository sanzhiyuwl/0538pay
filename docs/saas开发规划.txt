====================================================
0538pay 「控制台」SaaS 平台化开发规划
====================================================
定位：超级管理员 / 平台方视角。把 0538pay 支付系统作为服务出租——
      主站运营方创建、售卖、管理一个个「子站点租户（分站）」，
      每个分站是相对独立的支付站实例（自己的域名 / 管理员 / 商户 / 数据），
      按套餐 + 使用期限付费。这是彩虹易支付原生没有的能力，是本项目的 SaaS 核心增值。

术语：
  子站点租户 / 分站 = 一个独立的支付站实例（tenant / site）
  主站 / 平台方   = 0538pay 运营者（super admin），即控制台的使用者
  分站管理员      = 购买租户的下游，登录自己分站后台

导航：控制台作为独立一级入口，放左侧导航【最下方】，图标建议 LayoutGrid / Server。
      点击进入控制台独立页面（路由 /console，后续子功能用 /console/xxx）。

====================================================
一、功能模块规划（按优先级 P0>P1>P2）
====================================================

【P0 · v1 先做，本次实现】
1. 子站点租户管理（核心）
   - 分站列表：站点名称 / 域名 / 分站管理员 / 套餐 / 状态 / 到期时间 / 用量 / 操作
   - 状态：待激活 / 运行中 / 已停用 / 已过期
   - 创建租户：站点名称、绑定域名、管理员账号 + 初始密码、选择租户套餐、使用期限
   - 编辑租户 / 续期 / 停用·启用 / 删除
2. 权限分配
   - 每个分站可用的【功能模块】开关（交易/商户/支付接口/财务/风控/…）
   - 可用【支付通道】范围（哪些通道允许该分站使用）
   - 资源配额：最大商户数 / 最大通道数 / 月交易额上限
3. 使用期限
   - 开通时间 / 到期时间；到期自动停用；续费延长期限

【P1 · v2 规划】
4. 租户套餐 / 定价
   - 租户套餐（基础版 / 专业版 / 旗舰版），各版本：价格、期限、功能差异、配额差异
   - 与「用户组/套餐」区分：那个是分站内部对商户的套餐，这个是主站对分站的套餐
5. 租户计费 / 收入
   - 分站购买 / 续费租户的订单记录、主站从租户收入的账单
   - 与「账单中心」打通：租户售卖收入计入平台收入
6. 资源用量与超限告警
   - 各分站当前商户数 / 本月交易额 / 通道数 vs 配额，进度条展示，超限预警 / 自动限制

【P2 · v3 规划】
7. 跨分站数据总览
   - 全部分站的 GMV / 交易笔数 / 商户数 汇总看板，Top 分站排行，趋势对比
8. 分站运营工具
   - 一键登录任意分站后台（免密进入）、向分站下发公告、批量续期 / 停用
9. 分站独立配置 / 品牌定制
   - 独立域名绑定与 SSL、分站 Logo / 站点名 / 主题色定制（白标 white-label）
   - 数据隔离说明（每个分站数据独立，主站只做管理不看明细）
10. 操作审计
   - 控制台所有敏感操作（创建 / 停用 / 改权限 / 免密登录）留痕审计日志

====================================================
二、数据模型（mock 先行，字段对齐未来后端 pre_site 表设想）
====================================================
Site（子站点租户）:
  id            租户 ID
  name          站点名称
  domain        绑定域名（如 pay.xxx.com）
  adminUser     分站管理员账号
  plan          租户套餐（basic / pro / ultimate）
  status        0待激活 1运行中 2已停用 3已过期
  createTime    开通时间
  expireTime    到期时间
  permissions   功能模块权限（string[]，如 ['orders','merchants','channels',...]）
  channelScope  可用支付通道 ID 范围（number[]）
  quota         { maxMerchants, maxChannels, monthlyAmount } 配额
  usage         { merchants, monthlyAmount, channels } 当前用量

SitePlan（租户套餐，v2）:
  key / name / price / months / permissions / quota

====================================================
三、页面结构规划
====================================================
/console（控制台首页 = 子站点租户管理）
  - 顶部概况：租户总数 / 运行中 / 即将到期 / 已停用 / 平台累计收入
  - 工具栏：搜索（站点名/域名/管理员）+ 状态筛选 + 「创建租户」按钮
  - 租户列表：卡片 或 表格（含用量进度条）
  - 创建 / 编辑租户：Modal 或 抽屉（含权限勾选 + 配额 + 期限）

后续子页（v2/v3）：
  /console/plans     租户套餐定价
  /console/billing   租户计费收入
  /console/overview  跨分站数据总览
  /console/logs      控制台操作审计
