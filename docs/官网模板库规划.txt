====================================================
0538pay 官网模板库规划（多套首页模板 + 售卖给租户）
====================================================
目标：官网首页从「一套写死」升级为「多套可切换、可售卖的模板库」。
      结合 SaaS 控制台——租户/分站购买系统后，在分站后台选一套官网模板启用，
      模板可作为增值项售卖（免费/会员/付费解锁）。

决策（2026-07-14 确认）：
- 售卖对象：卖给租户/分站（呼应控制台「首页模板配置」）。
- 技术形式：组件式模板——每套 = 一个独立 Vue 组件，整体切换，精品质感。
- 现有处理：当前首页收编为首套「经典蓝 classic」，建 templates 目录统一管。

----------------------------------------
一、目录结构
----------------------------------------
admin-web/src/views/site/
  templates/                    ← 模板库根目录
    classic/                    ← 首套：经典蓝（现有首页迁入）
      ClassicHome.vue           ← 首页组件
      meta.ts                   ← 该模板元信息（可选，或集中在 registry）
    <next>/                     ← 后续每套一个文件夹
      XxxHome.vue
  registry.ts                   ← 模板注册表（统一元信息 + 组件映射）
  SiteHome.vue                  ← 壳：按当前启用模板 key 动态渲染对应模板组件

public/site/                    ← 模板图片素材（已建：hero/features/products/logos/misc）
  <每套模板的预览图>            ← 如 previews/classic.png

----------------------------------------
二、模板注册表（registry.ts）
----------------------------------------
每套模板登记：
- key（唯一标识，如 'classic'）
- name（展示名，如「经典蓝」）
- desc（一句话风格描述）
- preview（预览图路径 /site/previews/xxx.png）
- tier（免费 free / 会员 member / 付费 paid）—— 呼应售卖
- price（付费模板价格，free/member 为 0）
- component（异步 import 的模板组件，懒加载）

SiteHome.vue 作为壳：读取「当前启用模板 key」（原型阶段用默认值/本地状态，
未来由租户配置决定），渲染 registry 里对应的 component。

----------------------------------------
三、与售卖/控制台的衔接（分阶段）
----------------------------------------
阶段1（先做）：搭模板库底层 —— 目录 + registry + 壳，现有首页收编为 classic。
              只有一套时行为不变，用户访问 / 仍看到经典蓝。
阶段2：再做 1~2 套新模板组件（不同风格），验证「多套切换」跑通。
阶段3：模板市场页 —— 一个展示所有模板的画廊页（预览图 + 风格 + tier + 启用/购买按钮），
        可放控制台或官网。租户在此选用模板。
阶段4：接控制台「首页模板配置」——租户选中的模板 key 落到分站配置，
        分站官网据此渲染；付费模板走购买流程。

----------------------------------------
四、开发顺序与自检
----------------------------------------
1. 阶段1：建 templates/classic/ + registry.ts + 改 SiteHome 为壳，迁移现有首页。自检构建+真跑，行为不变。
2. 后续每加一套模板：新增组件 + registry 登记一行，自检。
3. 每步：vue-tsc / vite build / dev server 真跑。
