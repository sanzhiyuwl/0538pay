====================================================
0538pay 「商户中心」开发规划（对齐彩虹易支付 epay user 端）
====================================================
定位：商户（下游收款方）自己登录使用的自助端。区别于——
      · 管理后台 AdminLayout（运营方管全站，已完成）
      · SaaS 控制台 ConsoleLayout（平台方管租户/分站，已完成）
      本端是【商户视角】：查自己的订单/资金、提现、拿 API 密钥对接、实名、代付等。

对照基准：epay.test/user/（35 个 php 文件，Flatkit 紫色主题）已由三个只读子代理通读完毕（2026-07-14）。
本文档只做原型（mock 数据），不接后端；节奏同后台：先出高保真原型 → 满意 → 再接 API。

----------------------------------------
一、架构与挂载方式
----------------------------------------
- 复用现有 admin-web 工程（Vue3+TS+Vite+Tailwind+自建 shadcn 风组件），不新建工程。
- 新增第三个布局 MerchantLayout.vue，与 AdminLayout / ConsoleLayout 平级。
- 路由前缀 /m（如 /m、/m/orders、/m/api）。父路由挂 MerchantLayout，children 用相对路径。
- 导航配置加到 nav.ts：新增 merchantNav / merchantLeaves（一级平铺或分组，见第三节菜单树）。
- 视觉：不照搬 epay 的 Flatkit 紫色，沿用 0538pay 现代 SaaS 蓝色规范
  （直角无边框卡片 / Panel / .tbl 表格 / Badge / Modal / Drawer / Switch / field-input）。
- 登录/注册/找回密码是独立窄卡片页（无侧栏），单独路由（/m/login 等），不套 MerchantLayout。
- 复用现有组件：Panel/Button/Badge/Select/DateRange/Pagination/Modal/Drawer/Switch。
- mock 数据放 admin-web/src/lib/mock/merchant/ 下（或 merchant-*.ts），与后台 mock 分开。

----------------------------------------
二、商户中心与后台的区别（避免混淆）
----------------------------------------
- 后台 Orders：运营看【全站所有商户】订单，可改状态/退款/删除/补单。
  商户中心 Orders：只看【自己】的订单，操作限于 退款(需授权)/补单(重新通知)/看明细。
- 后台 Settle：运营审核/下发结算。商户中心 Settle：只看自己的结算记录 + 申请提现。
- 后台 Merchants：运营管所有商户账户。商户中心：只有自己的「账户设置」editinfo。
- 数据维度全部收敛到"当前登录商户 uid"。

注：条件显示项（保证金/提现/充值/代付/购买会员/邀请）在 epay 里受 $conf 开关控制，
    原型阶段全部做出来，用 mock 开关模拟"已开启"，页面顶部标注"该功能由平台开关控制"。

四、页面明细（字段以 epay 源码为准，逐页列关键项）
----------------------------------------

P0 · 核心闭环（先做，商户能登录→看数据→拿密钥→提现）
────────────────────────────────────────
[x] A1. MerchantLayout 布局 + 登录页 /m/login
    - 布局：顶栏（面包屑+主题+通知+用户菜单，用户菜单含 修改资料/API信息/修改密码/退出）
      + 左侧栏（按第三节菜单树，分组折叠，同 AdminLayout 风格）+ 内容区。
    - 登录页：密码登录(账号=邮箱/手机+密码) / 密钥登录(商户ID+商户密钥) 双 Tab；
      极验验证码占位；QQ/微信/支付宝快捷登录按钮；底部 找回密码/注册 入口。

[x] A2. 工作台首页 /m
    - 四数据卡片：商户当前余额 / 已结算余额 / 订单总数 / 今日订单。
    - 商户资料卡：头像+用户名+商户状态徽章（正常/已封禁/关闭支付/关闭结算/待审核/未实名）
      + 今日收入/昨日收入；右上"修改资料/API信息"快捷按钮。
    - 收入统计与通道费率表：按支付通道列 图标+名称/今日金额/昨日金额/成功率/费率。
    - 右侧：公告通知列表 + 结算金额趋势图（近 9 条结算，折线图，复用 chart.js）。
    - 顶部条件横幅：未实名强制提醒 / 未绑密保 / 未设登录密码（mock 开关控制）。

[x] A3. 订单记录 /m/orders
    - 筛选：搜索类型(系统单号/商户单号/接口单号/用户交易号/商品名/金额/域名/IP/支付账号)
      + 关键词 + 日期范围 + 支付方式 + 状态(全部/未付/已付/已退款/已冻结)。
    - 表格列：系统单号+商户单号 / 商品名+金额 / 实付 / 支付方式 / 创建+完成时间 / 状态 / 操作。
    - 操作：明细(跳资金明细) / 补单(重新通知) / 退款(弹窗填金额+登录密码，受 user_refund 开关)。
    - 统计弹窗：订单总/已付/未付/已退款金额、总收入利润、订单数、成功率。
    - 订单详情弹窗：完整字段 + 重新通知(异步/同步) + 子订单列表(合单)。

[x] A4. 资金明细 /m/records
    - 筛选：类型(操作类型/变更金额/关联订单号) + 关键词。
    - 表格列：操作类型(支出红/收入绿) / 变更金额(±) / 变更前 / 变更后 / 时间 / 关联订单号(可跳)。

[x] A5. 结算记录 /m/settle + 申请提现 /m/apply
    - 结算记录：筛选状态(全部/待结算/已完成/正在结算/结算失败)；
      列 结算方式(支付宝/微信/QQ/银行卡/机构，手动加[手动]) / 账号 / 金额 / 实际到账 / 时间 / 状态；
      操作 失败看原因 / 已完成确认收款(微信二维码)。
    - 申请提现：提现方式(只读) / 提现账号 / 姓名 / 当前余额 / 可提现余额 / 申请金额(可点全部)；
      规则提示 最低额/每日次数/手续费(min~max)/D+0·D+1；提交=提现申请。

[x] A6. API 信息 /m/api
    - 接口地址(只读复制) / 商户ID(只读)。
    - V1(MD5)：商户 MD5 密钥 + 重置按钮。
    - V2(RSA)：平台公钥 / 商户公钥 / 商户私钥(生成时一次性弹窗展示，不保存) + 生成/重置密钥对。
    - 签名方式开关：MD5+RSA 兼容 / 仅 RSA 安全。
    - V1/V2 开发文档入口。

[x] A7. 修改密码（右上角用户菜单 Modal，复用后台已做的 Modal 组件）
    - 登录密码：旧/新/确认。（商户端无支付密码，去掉支付密码段）

P1 · 账户与常用（第二批）
────────────────────────────────────────
[x] B1. 账户设置 /m/profile（editinfo，分板块卡片，每块独立保存）
    - 收款账号：结算方式(支付宝/微信/QQ/银行卡) + 收款账号 + 真实姓名。
    - 联系方式：手机(脱敏,短信验证改绑) / 邮箱 / QQ / 网站域名 / 密钥登录开关 /
      退款API开关 / 代付API开关 / 预留余额。
    - 消息提醒：新订单(+金额阈值)/结算/登录/投诉/违规/余额不足(+阈值)通知；
      渠道 微信公众号/邮件/短信/企业微信。
    - 扣费模式：余额扣费(经典) / 订单加费(买家承担)。
    - 第三方绑定：QQ/微信/支付宝 绑定/解绑。
    - 安全弹窗：验证密保/改密保/改手机（极验+验证码，原型走占位交互）。

[x] B2. 实名认证 /m/certificate
    - 三步进度条：填信息 → 扫码认证 → 完成。
    - 主体类型：个人(姓名+身份证) / 企业(公司名+营业执照+法人姓名+法人身份证)。
    - 认证方式(mock 可切)：支付宝扫码 / 支付宝跳转 / 微信扫码 / 手机三要素。
    - 认证费用提示；成功态：脱敏展示信息+认证时间+类型+方式；个人可升级企业。

[x] B3. 授权域名 /m/domains
    - 列表：域名 / 状态(正常绿/拒绝红/审核中蓝) / 提交时间 / 操作。
    - 添加域名弹窗(支持通配 *，待审核) / 删除。

[x] B4. 聚合收款码 /m/onecode
    - 收款链接(可复制) + 收款方名称(可保存)。
    - 收款码预览：多种模板风格切换 + 下载图片(canvas 合成，原型可用占位或简化)。

[x] C6. 使用说明 /m/help

五、开发顺序与自检
----------------------------------------
1. 先 A1（布局+登录）打通骨架，用户确认布局风格后再铺页。
2. 按 P0(A2~A7) → P1(B1~B4) → P2(C1~C6) → 登录态(D1~D3) 顺序逐页做。
3. 每完成一项：更新本文件状态位，按 self-check 流程自检（构建 vue-tsc / vite build / 规范 / 交互 / 真跑 dev server）。
4. 三处同步：nav.ts(merchantNav) → router 注册(/m 子路由) → mock 出数据。
5. 全部完成后统一并入 优化待办.txt（路由懒加载等）。
