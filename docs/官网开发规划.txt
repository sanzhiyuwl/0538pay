====================================================
0538pay 「官网」开发规划（营销官网，第四块）
====================================================
定位：对外营销门面 + 开发者文档 + 支付结果页。访客/潜在商户看到的第一屏。
      区别于 epay 的 11 套 template 模板——不照抄彩虹视觉，用 0538pay 自己的
      现代 SaaS 设计语言重做（信息骨架行业通用，视觉 100% 自研）。

对齐参考（仅信息结构，不抄视觉）：epay.test/template/default/（index/doc/agreement/payok/payerr）。

----------------------------------------
一、「不模仿」策略
----------------------------------------
- 信息架构行业通用（讲清 是什么/费率/渠道/怎么接入/立即注册），照做不算模仿。
- 视觉表达 100% 自研：沿用 0538pay design token（蓝主色 oklch(0.58 0.2 262)、
  直角/圆角 --radius、Inter 字体、lucide 图标、浅灰底白卡片），与后台/商户中心/控制台同源。
- 彩虹是十年前蓝紫渐变+拟物风；我们是现代 SaaS 简约风（大留白/清晰层级/克制配色/几何图标）。
- 决策已定（2026-07-14）：现代 SaaS 简约风 + 完整官网多页 + 并入 admin-web 加 /site 前缀。

----------------------------------------
二、架构与挂载
----------------------------------------
- 复用 admin-web 工程。新增 SiteLayout.vue（营销站布局：顶部透明/吸顶导航条 + 内容 + 大页脚），
  与 AdminLayout/ConsoleLayout/MerchantLayout 平级。
- 路由前缀 /site。首页 /site，其余 /site/docs、/site/agreement 等。
- 支付结果页 /site/payok、/site/payerr 是独立窄页（无导航条，或极简顶栏）。
- 导航条 CTA「登录/注册」跳商户中心 /m/login、/m/reg。
- 深浅色跟随现有 theme store。响应式（移动端汉堡菜单）。
- 组件尽量复用 Button/Badge/Card；营销专属区块（Hero/特性卡/价格表/步骤条）新写，放 components/site/。

----------------------------------------
三、页面清单
----------------------------------------
----------------------------------------
二·补. 11 套模板板块库汇总（2026-07-14 通读 default+index1~10）
----------------------------------------
高频必备（几乎每套都有）：顶部导航 / Banner首屏 / 特性优势卡(3~6) / 支付渠道墙 / 页脚。
中频：数据统计(常带数字滚动) / 接入流程步骤条 / 合作伙伴logo / 中段CTA长条 / 左右图文交替叙事。
低频加分：价格费率表(仅index9) / FAQ手风琴(仅index2) / 客户评价语录轮播(仅index7) /
          产品矩阵tab联动(index3:扫码/公众号/APP/小程序/H5/网关/跨境/企业收付款) / 团队介绍。
风格光谱：浅色扁平企业风(default/index2/3/4/5/8/10) / 深色科技风(index1/6/7/9) / 现代SaaS(index4 Argon最接近)。
epay 缺失、现代 SaaS 常见、可新增：客户评价、FAQ、产品矩阵、视频/截图展示。

我们的取舍：用现代 SaaS 简约风统一重构，采纳「导航/Hero/数据/特性卡/支付墙/费率表/流程/
左右图文场景/客户评价/FAQ/中段CTA/大页脚」这套最全骨架（比任一单套模板都完整），
视觉 100% 自研 token，动效用克制的进场过渡（不堆 particles/three.js）。

四、开发顺序与自检
----------------------------------------
1. 先 S1+S2（布局+首页）出风格，用户确认后再铺 S3~S7。
2. 每完成一项：自检（vue-tsc / vite build / 规范 / 交互 / 真跑 dev server）。
3. 三处同步：路由注册 /site → 导航锚点 → 内容/mock。
4. 全部完成后统一并入 优化待办.txt（路由懒加载）。
